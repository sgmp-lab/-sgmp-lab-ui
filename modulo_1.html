Laboratorio de Mejora de Procesos

Módulo integrado del SGMP para levantar, consolidar, analizar y transformar procesos. Incluye rutas de inicio, instrumentos previos, consolidación de JSON, estructura AS IS, análisis Lean Six Sigma, decisión AS IS vs TO BE, selección de mejoras y resultados finales.

Inicio del módulo

Selecciona si vas a iniciar un proceso dentro del LMP o si vas a continuar desde un archivo JSON previamente generado.

Paso 1

Selecciona una ruta de trabajo

Inicia un levantamiento nuevo, estructura documentacion existente o carga un respaldo JSON para continuar o actualizar el proceso.

Levantar proceso nuevo desde cero

Usa esta ruta cuando el proceso sera levantado mediante cuestionario, entrevista, observacion o captura manual.

Ruta esperada: Preparacion y contexto -> Carga y consolidacion.

Estructurar proceso existente o documentado

Usa esta ruta cuando ya existen SIPOC, Visio, PDF, PR, manuales, procedimientos, instructivos u otra evidencia documental previa.

Ruta esperada: Preparación y contexto -> Abrir Extractor SIPOC externo o cargar JSON generado -> Carga y consolidación.

Continuar o actualizar desde JSON

Carga un respaldo LMP guardado en cualquier etapa. Sirve para continuar un trabajo incompleto o actualizar un proceso ya finalizado.

Entrada esperada: JSON de respaldo LMP parcial o final.

Proceso existente o documentado

Esta ruta no procesa imágenes dentro del LMP. Usa el módulo externo Extractor SIPOC para convertir diagramas o SIPOC existentes en un JSON compatible y luego cárgalo aquí.

Ruta documental

Abrir Extractor SIPOC

Abre el módulo especializado para segmentar SIPOC, procesar recortes y generar el JSON compatible con LMP.

El Extractor funciona como módulo SGMP independiente: API automática, fallback manual y descarga de JSON final.

Cargar JSON del Extractor

Carga el JSON generado por el Extractor SIPOC. El LMP validará el contrato antes de incorporarlo como insumo del proceso.

Contrato esperado

El LMP aceptará el JSON si cumple el contrato SIPOC_EXTRACTOR_TO_LMP o si contiene una estructura equivalente con sipoc_extraction.

Decisión

Decide si el proceso cierra con AS IS o si continúa a TO BE. Esta decisión gobierna la siguiente etapa.

Paso 6

Cerrar con AS IS

Usa esta opción si el proceso debe terminar formalizado en su estado actual sin construir estado futuro.

El módulo pasará directo a resultados.

Continuar a TO BE

Usa esta opción si el proceso avanzará a evaluación de mejoras y definición del estado futuro.

Se habilitará la gestión de mejoras en la siguiente etapa.

One page de mejoras / TO BE

Evalúa cada mejora propuesta. El usuario decide si la ejecuta, la ajusta o la descarta. Solo las aceptadas o ajustadas alimentarán los resultados.
